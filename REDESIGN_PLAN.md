# 福委後台改版計畫

> 從「福委請款工具」→「全員福利入口」的架構轉型
> 日期：2026-06-23

---

## 一、訪談結論

| 項目 | 決定 |
|------|------|
| 使用對象 | 全體員工都能登入 |
| 請款範圍 | 只有福委會自己請款（員工不碰請款） |
| 角色 | 四種且**可重疊**：管理者、福委會(副委)、員工、HR |
| HR 職責 | 管按摩 / 福利預約 |
| 員工總覽 | 公告 + 我的按摩預約 + 午茶日 + 快速入口 |
| 副委總覽 | 待簽核 + 費用 + 午茶日 |
| 管理員 | 最大權限（所有人看到的 + 全部管理功能） |
| 手機導覽 | 按角色動態顯示 |
| 費用分析 | 移到請款模組裡 |
| 未來功能 | 員工旅遊等，半年內 |

## 二、核心設計決策：角色可重疊 → 用「能力(capability)」模型

因為一個人可能同時是 HR + 福委 + 申請人，角色**不能是單一字串**。

### 資料結構
```js
currentUser = {
  email: 'xxx@tsagroup.com.tw',
  name: '王小明',
  roles: ['employee', 'welfare', 'hr']   // ← 陣列，不是單一值
}
```

### 角色 → 能力對照（權限矩陣）

| 能力 (capability) | 員工 employee | 福委 welfare | HR hr | 管理者 manager |
|-------------------|:---:|:---:|:---:|:---:|
| 看總覽 / 公告 | ✅ | ✅ | ✅ | ✅ |
| 預約按摩 (本人) | ✅ | ✅ | ✅ | ✅ |
| 看午茶日 | ✅ | ✅ | ✅ | ✅ |
| 發公告 | | ✅ | ✅ | ✅ |
| 請款建單 | | ✅ | | ✅ |
| 請款審核 | | ✅ | | ✅ |
| 看費用分析 | | ✅ | | ✅ |
| 午茶日管理 | | ✅ | | ✅ |
| 按摩/福利管理 | | | ✅ | ✅ |
| 使用者/角色管理 | | | | ✅ |

> **權限取聯集**：一個人是 `['employee','welfare','hr']`，就同時擁有以上三欄所有打勾的能力。
> `manager` 是 superset，自動擁有全部能力。

### 程式判斷方式（統一入口）
```js
// 取代目前散落各處的 isAdmin / role 判斷
function can(capability) {
  const roles = (currentUser && currentUser.roles) || ['employee'];
  if (roles.includes('manager')) return true;          // 管理者全開
  return CAPABILITY_MAP[capability].some(r => roles.includes(r));
}
// 用法：if (can('reimbursement.approve')) { ... }
```

## 三、總覽頁改版：積木式區塊（依能力顯示）

不寫死「員工版/副委版/管理員版」三套，而是把總覽拆成多個區塊，
**有該能力才出現該區塊**——重疊角色自然組合。

| 區塊 | 顯示條件 |
|------|----------|
| 📢 公佈欄 | 全部人 |
| 💆 我的按摩預約（最近一筆 + 快速預約鈕） | 全部人 |
| ☕ 午茶日：下次舉辦 | 全部人 |
| ⚡ 快速入口（依能力過濾按鈕） | 全部人 |
| 📬 待審核請款（有幾件等我簽） | `can('reimbursement.approve')` |
| 💰 本月費用 / 預算摘要 | `can('analytics.view')` |
| 🗂️ 按摩/福利管理摘要（待審申請數） | `can('massage.manage')` |

> 目前的 4 張請款 stat 卡（總支出/進行中/已核准/需修正）會被上面這組取代；
> 純請款相關的數字收進「待審核請款」「本月費用」兩塊，且只有有權限的人看得到。

## 四、導覽列（側欄 + 手機底部）依角色動態顯示

| 入口 | 顯示條件 |
|------|----------|
| 總覽 | 全部人 |
| 按摩預約 | 全部人 |
| 午茶日 | 全部人 |
| 案件追蹤（請款） | `can('reimbursement.view')` |
| 請款審核 ← 原「待審核」改名 | `can('reimbursement.approve')` |
| 系統管理 | `can('admin.users')` |

> 手機底部最多放 5 個常用，其餘進「⋯ 更多」或側欄。
> 一般員工登入後底部只會看到：總覽、按摩預約、午茶日（乾淨清爽）。

## 五、其他調整

1. **費用分析圖表**：從總覽搬進「案件追蹤(請款)」模組內，只有有 `analytics.view` 的人看得到。
2. **「待審核」正名為「請款審核」**：避免跟按摩/福利的審核混淆。
3. **按摩預約 `isAdmin`**：改用 `can('massage.manage')`，不再 hardcode `true`。
4. **午茶日**：admin 判斷也收斂到 `can()`。
5. **系統管理頁**：使用者管理要能**指派多重角色**（勾選 employee/welfare/hr/manager）。
6. **公告發布權限**：`can('bulletin.post')`，福委/HR/管理者皆可。
